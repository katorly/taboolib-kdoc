Gets the XMaterial based on the material's ID (Magic Value) and data value. You should avoid using this for performance issues.

Return

Deprecated

this method loops through all the available materials and matches their ID using getId which takes a really long time. Plugins should no longer support IDs. If you want, you can make a Map cache yourself. This method obviously doesn't work for 1.13+ and will not be supported. This is only here for debugging purposes.

the ID (Magic value) of the material.
